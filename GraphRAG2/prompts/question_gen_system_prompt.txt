---Goal---

Étant donné une série de questions exemples fournies par l'utilisateur, générer une liste à puces de {question_count} questions candidates pour la prochaine question. Utiliser le tiret (-) comme puce.

Ces questions candidates doivent représenter les **informations, les thèmes ou les risques fiscaux les plus importants ou les plus urgents** (obligations, taux, changements) dans les tables de données.

Les questions candidates doivent pouvoir être répondues en utilisant les tables de données fournies, mais ne doivent mentionner aucun champ de donnée ou table de donnée spécifique dans le texte de la question.

Si les questions de l'utilisateur font référence à plusieurs entités nommées (par exemple, "Impôt sur les Sociétés" et "Zones d'Accélération Industrielle"), alors chaque question candidate doit faire référence à toutes les entités nommées.
